📚 InfoBot Documentation

Welcome to InfoBot! This guide will help you understand how to use the bot, what each command does, and how to interpret the information you receive.

What is InfoBot?

InfoBot is a powerful Discord and Telegram bot that helps you search and analyze blockchain data, including:

  • Wallet addresses and transactions
  • Farcaster profiles and casts
  • Zora creator coins and accounts
  • Clanker token deployments
  • Cross-chain transactions

Getting Started

To use InfoBot, you'll need to connect your Farcaster account. Use the /connect command in Discord or Telegram to get started.

Understanding Commands

InfoBot supports various commands to help you search and analyze different types of data. Each command is designed to provide specific information about the blockchain ecosystem.

Universal Search

/search <query> or /info <query> - This is your go-to command for finding information. It automatically detects what you're searching for (wallet, contract, username, etc.) and returns relevant results.

Specialized Commands

  • /zora <query> - Search Zora-specific data (accounts, contracts, creator coins)
  • /clanker <query> - Search Clanker token deployments
  • /casts <keyword> - Search Farcaster casts by keyword
  • /far <query> - Search Farcaster users
  • /w <query> - Wallet lookup (Ethereum or Solana)
  • /x <query> - Find Farcaster profile by X/Twitter handle
  • /relay <transaction> - Cross-chain transaction lookup

How to Read Results

When InfoBot returns results, you'll see various types of information:

  • Wallet Addresses: Shows the address, associated tokens, and transaction history
  • Farcaster Profiles: Displays username, FID, bio, and associated wallets
  • Token Information: Shows contract details, supply, and trading links
  • Transaction Data: Provides transaction hash, status, and related addresses

Interpreting Data

InfoBot provides raw blockchain data that you can use to make informed decisions. Always verify important information by checking the source on block explorers like BaseScan.

🔍 Clanker Coin Detection & Explanation

This document explains how to identify where a Clanker-compatible token was deployed (Clanker Platform, Farcaster, Bankr, or other affiliates) and how users can verify whether a deployment is real, legitimate, and properly attributed.

It also covers how to read deployment data, how to confirm creators, and how to investigate split-reward configurations—especially for Bankr deployments on X (Twitter).

1. Deployment Categories

Token deployments within the Clanker ecosystem currently fall into four main groups:

  1. Clanker Platform Deployment
  2. Farcaster Deployment
  3. Bankr Deployment (multiple methods)
  4. Other Clanker Affiliates

Each deployment method leaves different on-chain fingerprints that help users verify authenticity.

2. How to Verify Where a Token Was Deployed

Every Clanker-compatible token has a unique Clanker page and an on-chain creation transaction.

Example: https://clanker.world/clanker/<contractAddress>

To determine where a token was deployed:

  1. Visit the Clanker page.
    • Deployment metadata is often visible under "Creation Transaction" or "Deployment Config".
  2. Open the creation transaction on BaseScan via the link provided.
    • Example: https://basescan.org/tx/<txHash>
  3. Look for:
    • deploymentConfig.tokenConfig
    • deploymentConfig.rewardsConfig
    • Metadata containing context links (X post, cast URL, message ID, etc.).

This information is consistent across Clanker Platform, Bankr, and affiliate deployments.

3. Clanker Platform Deployments

Tokens deployed directly through the Clanker platform will usually show:

  • Clear metadata pointing to Clanker as the origin.
  • The deploying wallet's address in the creatorAdmin or creatorRewardRecipient fields.
  • No split accounts unless manually configured.

Clanker Platform deployments are usually the most straightforward: the deployer wallet = the creator reward wallet (unless the user registered additional reward wallets).

Clanker also allows multiple registered reward wallets, so you may see more than one reward recipient depending on the user's settings.

4. Farcaster Deployments

Deployments started through Farcaster (casts to integrations using Clanker-compatible bots or tools) typically include:

  • A Farcaster cast URL in the metadata.
  • A reference to the cast author.
  • A single reward address unless a split was configured in advance.

Farcaster-origin deployments behave similarly to Clanker deployments, but the initiating context is usually a cast ID instead of an X post.

5. Bankr Deployments

Bankr deployments are more complex because Bankr supports multiple input channels:

Bankr Deployment Methods

  • Sending a message to Bankr on the Base app
  • Messaging Bankr in Terminal
  • Tweeting @bankrbot on X
  • Casting at @bankrbot on Farcaster

All of these methods perform the same core deployment function, except for one special case:

X (Twitter) Reply Deployments — Split Rewards

Bankr has a special rule for deployments made as direct replies on X.

Bankr Announcement (paraphrased):

If a user deploys a token by replying to someone's original post, the token's fees are split 50/50 between the reply author (the deployer) and the original post author.

Source tweet: https://x.com/bankrbot/status/1904668350753771868

Example reply-based deployment:

  • Clanker page: https://clanker.world/clanker/0x4F4c8378900B9Bf5Dc8Ee724Fb99aAf50f786b8a
  • Related X post: https://x.com/ayowtfchil/status/1904668851305914804

6. How to Verify a Bankr Split Deployment

Open the creation transaction on BaseScan: https://basescan.org/tx/0x65660c6fd570ac34afced7e52c3b716a53f0f8d12675d1c957dd06ee26141e8f

Inside the decoded input, locate:

deploymentConfig.tokenConfig

This section often includes:

  • Metadata
  • Context links (X post URL, cast URL, message ID)
  • Deployment origin

deploymentConfig.rewardsConfig

This is the most important section for confirming authenticity.

Key fields:

creatorAdmin

  • The address that executed the deployment.
  • For X deployments from social accounts, this may be:
    • The Bankr admin wallet
    • A dead address (e.g., 0x000…dEaD) if the creator is not meant to control the token
    • A system address

creatorRewardRecipient

  • The wallet(s) receiving rewards from trading fees.
  • In a split deployment, this is a Split contract address, not a normal wallet.
  • Example: 0xa2Fe377b9a8432AF3707094A7252E415B859962D

If you open this address on Splits.org: https://app.splits.org/accounts/0xa2Fe377b9a8432AF3707094A7252E415B859962D/?chainId=8453

You will see two underlying wallets:

  • One belonging to the reply author (the person who deployed)
  • One belonging to the original post author

In the example case:

  • Wallet 1 → @hodljpegs (the deployer)
  • Wallet 2 → @elonmusk Bankr wallet (original post author)

How to Confirm These Identities

There are two common methods:

  1. Ask @bankrbot on X — Bankr will confirm the split participants.
  2. Search the wallet addresses on X — Many users have posted their Bankr wallet addresses publicly. This makes it possible to link the Split account back to real X accounts.

7. Understanding Legacy Behavior

Before the new X reply-split system:

  • X deployments only credited the deployer.
  • No automatic fee splits existed.
  • Clanker allowed users to manually register multiple reward wallets.

Both old and new approaches still appear today depending on how the token was deployed.

8. Summary of Verification Steps

To determine if a deployment is real and how it was created:

  1. Check the Clanker page
    • See if metadata indicates Clanker, Farcaster, Bankr, or another affiliate.
  2. Open the creation transaction
    • Look at:
      • tokenConfig → origin context
      • rewardsConfig → who receives rewards
  3. If rewardsConfig uses a Splits contract
    • It is likely a Bankr X reply deployment.
  4. Use Splits.org to inspect reward participants
    • Confirm who the two wallets are.
  5. Compare context links (X post, cast, etc.)
    • This verifies whether the deployment is legitimate.
